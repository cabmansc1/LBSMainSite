## Summary

- 41 pages with exactly 1 h1 (renders as 2 once header.php h1 counts)
- 2 pages with 0 h1 (rely on header.php's brand h1 — usually wrong)
- 4 pages with 2+ h1 in-file (renders as 3+ — definitely wrong)

## Proposed fixes

**Sitewide prerequisite:** demote `header.php`'s brand wordmark from `<h1>` to `<p class="brand-wordmark">`. Until that's done, *every* page renders with at least 2 h1s. Once done, the per-page counts in the table above become the rendered counts.

### Pages with no h1 in the file

Each of these needs an `<h1>` near the top of main content. If migrated to `seo_head.php`, output `<h1><?= $seo['h1'] ?? 'Fallback Title' ?></h1>`. Otherwise insert a literal h1 tied to the page topic.

- `find-your-ad.php`
- `neighborhood-card-checkout.php`

### Pages with multiple h1s in the file

In all four cases below, the **first** h1 is a hand-rolled brand wordmark ("Lowcountry Business Spotlight") baked into the page (not coming from `header.php`). The page-specific h1 is the second one — that should survive. The brand wordmark should be demoted to `<p class="brand-wordmark">` (or removed entirely if the page already includes `header.php`, which would duplicate it).

### Note on header.php's global h1

Phase 0 already flagged this: `header.php` wraps the brand wordmark in `<h1>`. **That fix is the highest-leverage single change** — it removes one h1 from every page that includes the partial. Until it ships, every page in the "ok" column above renders with 2 h1s, not 1.

When you make that change in `header.php`, also remove the hand-rolled brand wordmarks listed in the four files above (they currently bypass `header.php`).
